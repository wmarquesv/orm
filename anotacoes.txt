ORM (Object Relational Mapper)

E uma ferramenta que facilita o trabalho com banco de dados em aplicacoes, permitindo que seja feita interacao com o banco sem escrever consultas SQL diretamente.

Com um ORM, manipulamos tabelas e dados do banco como se fossem objetos com a linguagem de programacao, simplificando a leitura, escrita e manutencao do codigo. Ele tambem cuida da conversacao dos dados entre o banco de dados relacional
(tabelas) e os objetos no codigo.

Vantagens de utilizar um ORM

- Menos codigo SQL

- Facilita manutencao

- Compativel com multiplos banco de dados (se necessario a troca do db, se torna muito mais simples)

Apos as instalacao do prisma vamos executar o código de init para que ele nos retorne os arquivos desejados, sendo esse o codigo:

npx prisma init --datasource-provider postgresql

"postgresql" sendo o db desejado

ID UUID X INCREMENTAL

INCREMENTAL = sequencial de 1 em 1
id publico sendo muito mais facil de deduzir, sendo assim na mao de pessoas que entendem e querem fazer o mal pode encontrar falhas e fazer o que desejar

UUID = muitas regras/padrões para um id, tornando assim mais difícil a dedução do id

Para a criação de tabelas com o prisma, utilizamos o arquivo prisma.schema criado pelo nosso codigo de init.

model User{

}

model = modelo da tabelas
User = nome representativo da tabela

Dentro do objeto daremos os nomes e as atribuições as colunas

model User{
    id String @id @default(uuid())
    name String
    email String @unique

    @@map("users")
}

id = nome da coluna id | String = type string pois utilizaremos UUID | @id = para definir-la como primaria |  @default = para determinar valor padrão | (uuid()) = valor padrão uuid que sera gerado automaticamente

email @unique = para que cada usuario tenha um email unico

@@map = define o nome da tabela dentro do banco de dados

Apos a criacao do modelo precisamos da criacao real da tabela, fazemos isso com a seguinte sintaxe no terminal:

npx prisma migrate dev 

com o prisma criando uma pasta e um arquivo das nossas migrations desejadas com o codigo sql dentro dela

A criacao de um usuario no controller com o prisma segue a sintaxe dessa forma:

const { name, email } = request.body;

Recuperamos o name e email no body da nossa requisicao

await prisma.user.create({ data: { name, email } });

E esse e o método para criar um novo usuario

dentro do nosso prisma acessando nosso banco de dados (user) peço para criar (.create) referencio que os dados sao: (data:) e coloco os dados (:{name, email}) 

----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

A listagem de um usuario com o prisma segue a mesma simplicidade 

const users = await prisma.user.findMany();

A função findMany() encontrara todos da tabela

----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

A listagem de um usuário único e bem simples também

const { id } = request.params;

const user = await prisma.user.findUnique({ where: { id } });

Com o findUnique() precisamos apenas indicar onde sera baseado (where: id)

----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Para adicionarmos relacionamento entre as tabelas utilizamos essa sintaxe

user User @relation(fields: [userId], references: [id])

Criamos um usuário com a tipagem (User) e adicionamos uma relação (@relation()) no fields definimos quem sera a chave estrangeira, no references a chave primaria da coluna desejada (lembrando que a tipagem (User) declara qual tabela e)

E preciso declarar tambem que a nossa tabela de Usuario que ela se conecta com a tabela de questoes, com a seguinte sintaxe:

questions Questions[]

Deixando como Array[] declaramos que e uma conexao de um pra muitos = 1 User = Muitas perguntas

Caso seja 1:1 poderiamos declarar apenas 

questions Questions

Sem tornar-la uma Array

----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Para adicionarmos filtro na nossa listagem inves do like ou %palavra desejada% anteriormente, utilizamos a sintaxe dessa forma:

const questions = await prisma.questions.findMany({
      where: {
        title: {
          contains: request.query.title?.toString(),
          mode: "insensitive",
        },
      },
    });

dentro do nosso método de listagem, adicionamos o (where) para pedir direção, usando o (title) como parâmetro, inves de procurar-mos por idêntico(exatamente igual), utilizamos o contains para substituir o %% ou like do sql sendo ele
opcional (?.toString().trim()) caso nao queiram fazer a filtragem, adicionamos tambem o (mode: "insensitive") para que ele nao seja case sensitive fazendo assim sua filtragem perante a letra, e nao pela letra exatamente igual

----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Para atualizar uma questão utilizaremos esse comando:

const { id } = request.params;
    const { title, content } = request.body;

    await prisma.questions.update({
      data: { title, content },
      where: { id },
    });

Recuperamos o id da pergunta, o titulo e o conteudo da pergunta, especificamos que queremos atualizar (.update) o conteudo e o titulo(data: {title,content}) do id recuperado (where:{ id })
